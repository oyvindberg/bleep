# bleep run

Run a project's main class or a script.

## Synopsis

```bash
bleep run [options] <project-or-script> [-- args...]
```

## Description

The `run` command compiles and executes a project's main class, or runs a registered script. Arguments after `--` are passed to the running program.

## Arguments

| Argument | Description |
|----------|-------------|
| `project-or-script` | Name of the project to run, or name of a script |
| `args` | Arguments to pass to the program (after `--`) |

## Options

Accepts all [global options](/docs/reference/cli/overview#global-options).

## Examples

### Run a project

```bash
bleep run myapp
```

### Run with arguments

```bash
bleep run myapp -- --config prod.conf --verbose
```

### Run a script

```bash
bleep run publish
```

Or simply:

```bash
bleep publish
```

(Scripts registered in `bleep.yaml` can be run directly as subcommands)

## Main Class Configuration

The main class is specified in your project's platform configuration:

```yaml
projects:
  myapp:
    platform:
      mainClass: com.example.Main
```

If a project has multiple main classes, you must specify which one to use.

## Running Scripts

Scripts are custom commands defined in `bleep.yaml`:

```yaml
scripts:
  my-script:
    main: scripts.MyScript
    project: scripts
```

Scripts appear as top-level commands:

```bash
bleep my-script
```

See [Write Your First Script](/docs/tutorials/your-first-script) for details.

## JVM Options

To pass JVM options, use the `jvmOptions` field in your project configuration:

```yaml
projects:
  myapp:
    platform:
      mainClass: com.example.Main
      jvmOptions:
        - -Xmx2g
        - -XX:+UseG1GC
```
