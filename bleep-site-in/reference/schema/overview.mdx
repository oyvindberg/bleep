# bleep.yaml Schema

The `bleep.yaml` file defines your entire build configuration.

## Basic Structure

```yaml
$schema: https://raw.githubusercontent.com/oyvindberg/bleep/master/schema.json
$version: 0.0.14

jvm:
  name: graalvm-community:21

templates:
  # Shared configuration

projects:
  # Your projects

scripts:
  # Custom commands
```

## Top-Level Fields

| Field | Required | Description |
|-------|----------|-------------|
| `$schema` | No | JSON Schema URL for IDE validation |
| `$version` | Yes | Bleep version to use |
| `jvm` | No | JVM configuration |
| `templates` | No | Reusable configuration templates |
| `projects` | Yes | Project definitions |
| `scripts` | No | Custom script commands |

## Schema Validation

The `$schema` field enables IDE autocompletion and validation:

```yaml
$schema: https://raw.githubusercontent.com/oyvindberg/bleep/master/schema.json
```

## JVM Configuration

```yaml
jvm:
  name: graalvm-community:21
```

Bleep manages JVM downloads via Coursier. Common options:
- `graalvm-community:21`
- `temurin:21`
- `temurin:17`
