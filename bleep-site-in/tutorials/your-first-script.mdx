# Write Your First Script

Bleep scripts are regular Scala programs that can interact with your build. Unlike sbt plugins or tasks, scripts are just code you can debug, test, and run like any other program.

## What Are Scripts?

In sbt, custom build logic is expressed through tasks and plugins with a specialized DSL. In bleep, you write regular Scala code in a dedicated project and register it as a script.

Scripts can:
- Access build information (projects, dependencies, versions)
- Run build commands (compile, test, run)
- Generate code or resources
- Publish artifacts
- Anything else you can do in Scala

## Prerequisites

- A working bleep project (see [Your First Project](/docs/tutorials/your-first-project))
- Basic Scala knowledge

## Step 1: Create a Scripts Project

Add a scripts project to your `bleep.yaml`:

```yaml
$schema: https://raw.githubusercontent.com/oyvindberg/bleep/master/schema.json
$version: 0.0.12

jvm:
  name: graalvm-community:21

projects:
  myapp:
    scala:
      version: 3.3.3
    platform:
      mainClass: myapp.Main

  scripts:
    dependencies:
      - build.bleep::bleep-core:${BLEEP_VERSION}
    scala:
      version: 3.3.3

scripts:
  hello:
    main: scripts.HelloScript
    project: scripts
```

Key points:
- The `scripts` project depends on `bleep-core` to access build APIs
- `${BLEEP_VERSION}` automatically uses your installed bleep version
- The `scripts:` section registers the script with bleep

## Step 2: Write the Script

Create the directory structure:

```bash
mkdir -p scripts/src/scala/scripts
```

Create `scripts/src/scala/scripts/HelloScript.scala`:

```scala
package scripts

import bleep._
import bleep.commands.Script

object HelloScript extends Script("HelloScript"):
  override def run(started: Started, commands: Commands, args: List[String]): Unit =
    // Access build information
    val projectCount = started.build.explodedProjects.size
    started.logger.info(s"This build has $projectCount projects")

    // List all project names
    started.build.explodedProjects.keys.foreach: projectName =>
      started.logger.info(s"  - ${projectName.value}")

    // Handle command-line arguments
    args match
      case name :: Nil =>
        started.logger.info(s"Hello, $name!")
      case Nil =>
        started.logger.info("Hello, world!")
      case _ =>
        started.logger.error("Usage: bleep hello [name]")
```

## Step 3: Run the Script

```bash
bleep hello
```

Output:
```
[info] This build has 2 projects
[info]   - myapp
[info]   - scripts
[info] Hello, world!
```

With an argument:

```bash
bleep hello Alice
```

Output:
```
[info] This build has 2 projects
[info]   - myapp
[info]   - scripts
[info] Hello, Alice!
```

## Understanding the Script API

### The `Started` Object

The `Started` object gives you access to:

```scala
// The resolved build model
started.build                  // BuildModel
started.build.explodedProjects // Map[ProjectName, Project]

// Pre-resolved project directories
started.projectPaths          // Map[ProjectName, ProjectPaths]

// Logger for output
started.logger                // Logger

// Build directory information
started.buildPaths            // BuildPaths
```

### The `Commands` Object

The `Commands` object lets you invoke build commands:

```scala
// Compile specific projects
commands.compile(List(model.ProjectName("myapp")))

// Run tests
commands.test(List(model.ProjectName("myapp-test")))

// Publish locally
commands.publishLocal(List(model.ProjectName("myapp")))
```

## Practical Example: Code Generator

Let's create a script that generates a version file:

```scala
package scripts

import bleep._
import bleep.commands.Script
import java.nio.file.Files

object GenerateVersion extends Script("GenerateVersion"):
  override def run(started: Started, commands: Commands, args: List[String]): Unit =
    val version = started.build.explodedProjects
      .get(model.ProjectName("myapp"))
      .flatMap(_.version)
      .getOrElse("0.0.1-SNAPSHOT")

    val targetProject = model.ProjectName("myapp")
    val projectPath = started.projectPaths(targetProject)

    // Generate to a "generated" directory within the project
    val genDir = projectPath.dir / "generated" / "scala" / "myapp"
    Files.createDirectories(genDir.toNIO)

    val versionFile = genDir / "BuildInfo.scala"
    val content = s"""package myapp
                     |
                     |object BuildInfo:
                     |  val version: String = "$version"
                     |  val scalaVersion: String = "${started.build.explodedProjects(targetProject).scala.flatMap(_.version).getOrElse("unknown")}"
                     |""".stripMargin

    Files.writeString(versionFile.toNIO, content)
    started.logger.info(s"Generated $versionFile")
```

Register it in `bleep.yaml`:

```yaml
scripts:
  hello:
    main: scripts.HelloScript
    project: scripts
  generate-version:
    main: scripts.GenerateVersion
    project: scripts
```

To include the generated sources, update your main project:

```yaml
projects:
  myapp:
    sources:
      - generated/scala
    # ... rest of config
```

## Debugging Scripts

Scripts are regular Scala programs, so you can debug them in your IDE:

1. Run `bleep setup-ide` to generate IDE configuration
2. Open the project in IntelliJ or VS Code with Metals
3. Set breakpoints in your script
4. Create a run configuration that executes your script's main class

## Scripts vs sbt Plugins

| Aspect | sbt Plugins | Bleep Scripts |
|--------|-------------|---------------|
| Language | Scala DSL with macros | Plain Scala |
| Debugging | Difficult | Normal IDE debugging |
| Dependencies | `addSbtPlugin` in project/ | Regular dependencies |
| Execution | Via task graph | Direct invocation |
| Caching | Automatic | Manual if needed |
