# Projects & Dependencies

This page explains how projects and dependencies work in bleep.

## Projects

A project in bleep represents a compilation unit - a set of source files that compile together to produce a JAR file or executable.

### Basic Project Structure

```yaml
projects:
  myapp:
    scala:
      version: 3.3.3
    platform:
      mainClass: com.example.Main
```

Each project has a corresponding directory with the same name containing source files:

```
myproject/
├── bleep.yaml
└── myapp/
    └── src/
        └── scala/
            └── com/
                └── example/
                    └── Main.scala
```

### Source Directories

By default, bleep looks for sources in:
- `{project}/src/scala/` - Scala source files
- `{project}/src/java/` - Java source files
- `{project}/resources/` - Resource files

You can customize source locations:

```yaml
projects:
  myapp:
    sources:
      - src/main/scala
      - generated/scala
    resources:
      - src/main/resources
```

### Project Dependencies

Projects can depend on other projects in the same build:

```yaml
projects:
  core:
    scala:
      version: 3.3.3

  app:
    dependsOn: core
    scala:
      version: 3.3.3
```

Multiple dependencies:

```yaml
projects:
  app:
    dependsOn:
      - core
      - utils
```

## External Dependencies

External dependencies are libraries from Maven repositories.

### Scala Dependencies

Scala dependencies use `::` (double colon) to automatically append the Scala version:

```yaml
dependencies:
  - com.lihaoyi::fansi:0.5.0
```

This resolves to `fansi_3` for Scala 3 or `fansi_2.13` for Scala 2.13.

### Java Dependencies

Java dependencies use `:` (single colon):

```yaml
dependencies:
  - org.slf4j:slf4j-api:2.0.9
```

### Full Coordinates

You can also specify full coordinates:

```yaml
dependencies:
  - org.typelevel:cats-core_3:2.10.0
```

### Dependency Attributes

Specify classifiers, exclusions, or configurations:

```yaml
dependencies:
  - module: org.lwjgl:lwjgl:3.3.1
    classifier: natives-macos
  - module: com.example::mylib:1.0.0
    exclusions:
      - org.slf4j:*
```

### Version Ranges

Bleep supports Maven version ranges:

```yaml
dependencies:
  - com.lihaoyi::fansi:[0.4.0,0.6.0)
```

However, fixed versions are recommended for reproducibility.

## Dependency Scopes

### Compile Dependencies

Regular dependencies are available at compile time and runtime:

```yaml
projects:
  myapp:
    dependencies:
      - com.lihaoyi::fansi:0.5.0
```

### Test Dependencies

Test-only dependencies go in test projects:

```yaml
projects:
  myapp-test:
    isTestProject: true
    dependsOn: myapp
    dependencies:
      - org.scalameta::munit:1.0.0
```

### Provided Dependencies

For dependencies that are provided at runtime (like servlet APIs):

```yaml
dependencies:
  - module: javax.servlet:javax.servlet-api:4.0.1
    configuration: provided
```

## Resolving Dependencies

Bleep uses Coursier for dependency resolution. Dependencies are resolved on first compile or when `bleep.yaml` changes.

View resolved dependencies:

```bash
bleep build show
```

Update to latest versions:

```bash
bleep build update-deps
```

## Dependency Conflicts

When multiple versions of a library are requested, Coursier picks the highest version by default. For stricter control, you can force specific versions:

```yaml
projects:
  myapp:
    dependencies:
      - com.lihaoyi::fansi:0.5.0
    # Force a specific version of a transitive dependency
    dependenciesOverride:
      - org.scala-lang:scala-library:2.13.12
```
