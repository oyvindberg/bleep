# `bleep config`

<p>manage user-level bleep configuration: auth, compile-server / test-runner JVM settings, remote-cache credentials, log timing</p>

## Synopsis

```bash
bleep config <subcommand> [args] [flags]
```

## Subcommands

- [`bleep config file`](./file) &mdash; show configuration file location
- [`bleep config log-timing-enable`](./log-timing-enable) &mdash; enable timing info in logs
- [`bleep config log-timing-disable`](./log-timing-disable) &mdash; disable timing info in logs
- [`bleep config compile-server`](./compile-server) &mdash; You can speed up normal usage by keeping the compile server running between invocations. This is where you control it
- [`bleep config test-runner`](./test-runner) &mdash; configure test runner JVM settings
- [`bleep config auth`](./auth) &mdash; configure authentication for private repositories (Artifact Registry, GitHub Packages, GitLab)
- [`bleep config remote-cache`](./remote-cache) &mdash; configure remote build cache credentials
